# LanTalk 测试计划

## 阶段 1
- `dotnet build LanTalk.sln`
- `dotnet test LanTalk.sln`
- `dotnet run --project src/LanTalk.App`
- 验证主窗口可启动，左侧显示本机信息、搜索框、广播入口、最近会话和按部门/分组展示的真实在线用户/已知联系人。
- 验证左侧本机信息显示 IP 与在线状态；初始化失败时状态栏能显示明确错误。
- 在搜索框输入昵称、IP、最近消息或状态关键字，确认最近会话和在线用户列表同步过滤。
- 打开任意会话，在右侧聊天记录搜索框输入关键词，确认只显示当前会话中匹配的历史消息；清空关键词后恢复最近消息。
- 验证右侧显示聊天标题、消息气泡、文件消息卡片和输入栏。
- 验证设置面板可打开，昵称、部门/分组、文件目录、保存历史、主题和端口信息可见。
- 修改部门/分组后保存并重启，确认本机信息和对端联系人分组同步更新。
- 修改 UDP/TCP 消息/TCP 文件端口，确认端口范围为 1024-65535，重复端口不能保存，合法端口保存后提示重启生效。
- 逐条添加/删除自动发现网段，确认 `Auto`、`192.168.1.0/24`、`192.168.1.*`、`192.168.1.255` 可保存，非法格式不能保存。
- 点击窗口关闭按钮，确认窗口隐藏到系统托盘；点击托盘图标或菜单“打开 LanTalk”可恢复窗口。
- 使用托盘菜单“退出”，确认 UDP/TCP 监听关闭并退出进程。
- 在窗口隐藏、最小化或非活跃时接收私聊、广播或文件请求，确认桌面通知弹出，窗口标题和托盘提示显示未读数量。
- 收到多个会话消息后，确认最近会话按未读优先、最近活跃时间排序。
- 使用“上一个”“下一个”“未读”按钮或 `Ctrl+Tab`、`Ctrl+Shift+Tab`、`Ctrl+U` 切换会话，确认消息列表和未读数同步刷新。
- 点击“新建群组”，选择在线或已知联系人，确认可创建临时群组和永久群组。
- 在群组会话中发送文本消息，确认所有在线成员收到群组会话和未读提醒；关闭后重新打开永久群组仍存在。
- 在群组会话勾选端到端加密，确认在线成员自动发起一对一加密协商；协商完成成员收到群文本，未完成协商或离线成员进入加密补发队列且不收到明文。
- 点击表情按钮，选择常用表情，确认表情追加到输入框并可随文本发送。
- 在私聊会话勾选端到端加密，确认两端状态变为已启用且指纹一致；发送文本后接收方能看到明文内容，关闭后恢复普通私聊。
- 点击图片按钮选择 PNG/JPG/GIF/BMP/WebP 图片，确认聊天中生成图片消息卡片；接收方确认后图片可预览。
- 点击图片缩略图，确认打开大图预览；使用放大、缩小、原始和关闭按钮验证缩放状态。
- 验证联系人列表、本机信息和最近会话头像显示稳定首字与色彩。
- 点击文件按钮一次选择多个文件，确认接收端只弹出一次确认框，接受后所有文件依次传输并显示总体进度。
- 点击文件夹按钮选择包含子目录的文件夹，确认接收端保存后保留相对目录结构，空文件夹可创建。
- 传输过程中断开接收端或关闭网络，再重新发送同一文件/批次，确认接收端已有文件会触发断点续传，发送端从已有字节数继续传输。

## 自动化测试
当前测试覆盖：
- JSON Source Generator 序列化。
- 设置创建、保存、损坏恢复。
- 设置端口保存、非法端口规范化、重复端口规范化。
- 自动发现网段解析、广播地址计算、非法网段回退。
- SQLite 初始化。
- KnownUsers 旧表自动迁移部门字段。
- 设置、发现包、KnownUsers 的部门字段保存与反序列化兼容。
- 图片文件请求的 `IsImage` 字段新旧协议兼容。
- 群组文件请求元数据可随 `FileTransferRequest` 序列化，并兼容旧文件请求。
- 多文件/文件夹批次元数据和文件项可随 `FileTransferRequest` 序列化。
- `FileTransferResponse` 可携带单文件 `ResumeOffset` 和批次 `ResumeItems`。
- 图片消息元数据可使用 `LanTalkJsonContext` 序列化。
- `MessageRepository` 可保存和读取 `MessageKind.Image` 历史消息。
- `MessageRepository` 可保存和读取 `MessageKind.Group` 历史消息。
- `GroupRepository` 可保存、加载、删除永久群组，并忽略临时群组持久化。
- `OutgoingDeliveryRepository` 可保存、更新重试信息、删除待补发群组消息/附件，并保留附件源文件路径和加密补发要求。
- 当前会话聊天记录搜索，包含会话隔离和 `%` 等特殊字符查询。
- 文件传输记录保存，包含传输类型、批次 ID、相对路径和已传输字节数。
- `FileTransfers` 旧表可自动迁移批次与断点续传字段。
- 在线用户注册表。
- TCP 私聊回环。
- 端到端加密载荷序列化、ECDH/AES-GCM 加解密、加密私聊回环和加密群组消息回环。
- 群组载荷序列化、`GroupMessage` TCP 回环和群组文件请求 TCP 回环。
- 广播部分失败统计。
- TCP 文件流式传输与断点续传。

运行：
```bash
dotnet test LanTalk.sln
```

## 后续阶段
- UDP 自动发现：两台或三台电脑互相发现，关闭后离线，重新上线恢复。
- 自定义网段发现：设置为实际局域网 CIDR 后重启，确认可以发现同网段设备。
- 联系人分组：两端设置不同部门，确认 UDP 发现后按部门分组展示；旧配置或旧发现包应回落到“默认部门”。
- TCP 私聊：A 发给 B，B 回复 A，重启后加载历史。
- 群组：A 创建临时群组和永久群组并发送文本；B/C 在线时收到群组消息；重启后永久群组保留、临时群组不保留。
- 群组图片/文件：A 在群组会话发送图片或文件；B/C 在线成员分别收到来自群组的接收确认弹窗；B 接收、C 拒绝时，A 端群组附件卡片显示完成/拒绝/失败汇总。
- 群组文本离线补发：A 给含 B/C 的群组发文本时 C 离线，A 端提示进入离线补发；C 重新上线后确认收到该群组消息。
- 群组附件离线补发：A 给含 B/C 的群组发图片或文件时 C 离线，A 端提示进入离线补发；C 重新上线后收到附件接收请求；若 A 本地源文件被删除，A 端应保留队列并提示源文件不可用。
- 群组端到端加密：A 在群组会话开启加密，B 在线并完成协商后收到加密群文本；C 离线或未完成协商时进入加密补发队列；C 上线并完成协商后自动补发，且补发前不降级为明文。
- 端到端加密：A 在与 B 的私聊中启用加密，B 自动完成协商；两端比对指纹一致后互发文本；任一方关闭开关后确认后续文本仍可发送。
- 广播：A 发广播，B/C 收到，样式与私聊区分。
- 文件传输：小文件、100MB 文件、多文件、文件夹、拒绝接收、传输中断后断点续传、保存路径异常、UI 不冻结。
- 文件夹安全：构造包含绝对路径或 `..` 的异常批次请求，确认接收端拒绝，不会写出文件接收目录。
- 图片消息：发送图片、拒绝图片、接收图片、接收完成后预览/缩放、重启后本地图片仍在时可从历史中预览。
- 端口占用：启动前占用 50000/50001/50002 任一端口，确认应用给出明确初始化失败提示。
- 聊天记录搜索：在私聊和广播会话中分别搜索关键词，确认结果只来自当前会话。
- 托盘/通知：隐藏到托盘后接收消息，确认通知弹出；点击通知可恢复窗口；打开对应会话后未读数清零。
